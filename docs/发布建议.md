# 发布建议

## 适合 GitHub 的方案

最推荐：

- 当前这个索引库直接发到 GitHub
- 后续开启 GitHub Pages，做成可浏览的纪念站

优点：

- 免费
- 便于版本管理
- 容易接受他人补充 Pull Request
- 适合长期维护目录、链接和说明文字

限制：

- 不适合直接托管大量视频文件
- 对受版权保护内容不友好

## 如果你想做更完整的展示页

可以再加一层静态站：

- VitePress
- Astro
- Hugo

适合做：

- 首页纪念文案
- 时间线
- 分类浏览
- 视频卡片墙
- 图书页

## 如果你想保存媒体原件

建议不要直接放 GitHub 仓库。

更合适的方式：

- 对象存储
- 私有网盘
- NAS
- 只读备份盘

公开页面只引用你确认能公开的内容。

## 推荐组合

对于你的目标，最实用的组合是：

1. GitHub 仓库：放目录、说明、截图、索引
2. GitHub Pages：放公开纪念页面
3. 私有备份：放你自己保存的原始文件

这样既能公开纪念，也能保留后续扩充空间。
