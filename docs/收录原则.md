# 收录原则

## 1. 这个仓库收什么

- 官方账号主页
- 代表视频的原始链接
- 图书、课程、专栏等公开出版物信息
- 重要采访、争议事件、公益内容、人物节点
- 自己写的纪念说明、摘要、标签、时间线

## 2. 这个仓库默认不收什么

- 未经授权的整本电子书
- 批量搬运的完整视频文件
- 平台受限下载内容的再分发包
- 来路不明的网盘资源

## 3. 为什么这样做

如果你准备把项目发到 GitHub，最稳妥的方式不是“资源下载站”，而是“纪念索引库”：

- 公开性强
- 可持续补录
- 风险更低
- 更容易获得他人协作补档

## 4. 推荐的数据补录标准

每条内容尽量补齐这些字段：

- 标题
- 平台
- 原链接
- 发布时间
- 是否官方
- 内容主题
- 一句话说明
- 核验状态

## 5. 后续优先补录的内容

- 2016 年前后爆火的讲座原始版本
- 经典直播切片的更早出处
- 微博长文、视频号、采访节目
- 图书封面、目录、出版信息截图
- 综艺、演讲、访谈节目单

## 6. 如果你确实要做“媒体备份版”

建议拆成两个仓库或两套存储：

- 公开仓库：只放索引、摘要、截图、合法引用
- 私有存储：你自己保存的原始媒体文件

这样公开版可以长期存在，私有版再按你的授权情况决定是否公开。
